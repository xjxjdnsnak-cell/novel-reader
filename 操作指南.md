# Novel Reader 操作指南

这份指南面向日常使用：导入小说、连接本地 Qwen embedding、在 Claude Code 里检索/问答/分析/续写。

## 0. 最简单方式：打开本地 Web 操作台

如果你不想记命令，优先用网页端。它可以完成导入、查看书籍、阅读章节、搜索、情节问答证据包、分析/地图、风格蒸馏、续写任务包和 embedding 状态检查。

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader
pip install -e .
.\bin\start-web.ps1 -OpenBrowser
```

打开地址：

```text
http://127.0.0.1:8765
```

如果你已经启动了 Qwen Embedding，网页会自动识别 `http://127.0.0.1:8081/v1`。如果没有启动，也不影响关键词检索、阅读、问答证据包、风格蒸馏和续写任务包。

如果你之前已经保存过模型路径，或者模型在魔搭默认缓存位置，网页启动脚本会自动尝试启动 Qwen Embedding。也可以手动指定模型路径：

```powershell
.\bin\start-web.ps1 -ModelPath "C:\Users\xsjhxs\.cache\modelscope\hub\models\Qwen\Qwen3-Embedding-0.6B" -OpenBrowser
```

只启动网页、不使用 embedding：

```powershell
.\bin\start-web.ps1 -NoEmbedding -OpenBrowser
```

先启动 Qwen Embedding，再打开网页：

```powershell
.\bin\start-novel-reader.ps1 -Client none
.\bin\start-web.ps1 -OpenBrowser
```

网页服务默认只监听本机 `127.0.0.1`，不会主动暴露到局域网。

## 0.1 两种 Claude 使用模式

### 模式一：打开 Claude 插件模式

这个模式适合你想主要在 Claude Code 里工作，让 Claude 使用 Novel Reader 插件命令。

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader
.\bin\start-claude-plugin.ps1
```

带 Qwen Embedding：

```powershell
.\bin\start-claude-plugin.ps1 -ModelPath "C:\Users\xsjhxs\.cache\modelscope\hub\models\Qwen\Qwen3-Embedding-0.6B"
```

危险权限模式：

```powershell
.\bin\start-claude-plugin.ps1 -ClaudePermissionMode dangerous
```

脚本会要求输入 `DANGEROUS` 后才会继续。

### 模式二：在 Claude 里呼出前端面板

这个模式适合你已经在 Claude Code 里，希望输入命令直接打开 Novel Reader 网页面板。

在 Claude Code 里输入：

```text
/novel-books
/novel-books use <book_id>
/novel-web
```

可选：

```text
/novel-web --port 8770
/novel-web --no-embedding
/novel-web --dangerous
```

`/novel-web` 会在后台启动 Web 操作台、打开浏览器，并启用网页里的 Claude bridge。如果端口被旧服务占用，它会自动换到下一个可用端口，并在 Claude 输出里告诉你最终地址。

### 网页里查看生成文档

生成“情节梳理 / 全书地图 / 写作分析 / 风格文档 / 续写任务包”后，打开网页里的“文档”页签。

你可以：

```text
预览 Markdown
查看源码
下载文件
```

网页只允许读取当前书籍目录下这些产物目录：

```text
maps
reports
styles
continuations
summaries
```

不会读取任意本地文件。

### 网页里连接 Claude Code

默认网页不会启用 Claude 聊天。需要你启动时显式打开：

```powershell
.\bin\start-web.ps1 -EnableClaudeChat -OpenBrowser
```

网页里的 Claude 页签支持：

```text
独立调用：每次发送都是一次新请求
连续会话：使用 claude -c -p 继续 Claude Code 会话
附加当前文档
附加问答证据包
附加风格证据包
附加续写任务包
```

如果你要启用危险权限模式：

```powershell
.\bin\start-web.ps1 -EnableClaudeChat -ClaudePermissionMode dangerous -OpenBrowser
```

脚本会要求输入：

```text
DANGEROUS
```

确认后才会给 Claude CLI 加上 `--dangerously-skip-permissions`。这个模式风险更高，只建议在你信任当前项目目录时使用。

## 3. 启动本地 Qwen Embedding 服务

推荐先用一键启动脚本。它会自动检查 Qwen 模型、启动 embedding 服务，并带环境变量启动 Claude Code：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

.\bin\start-novel-reader.ps1
```

启动时会让你选择：

```text
1. Normal Claude
2. Claude --dangerously-skip-permissions
```

默认建议选 1。选 2 时脚本会要求输入 `DANGEROUS` 二次确认。

如果不想每次选择，可以直接指定：

```powershell
.\bin\start-novel-reader.ps1 -ClaudePermissionMode normal
.\bin\start-novel-reader.ps1 -ClaudePermissionMode dangerous
```

如果 Windows 提示禁止运行脚本，用这个命令启动：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\bin\start-novel-reader.ps1
```

启动 OpenCode：

```powershell
.\bin\start-novel-reader.ps1 -Client opencode
```

只启动或检查 embedding 服务，不启动客户端：

```powershell
.\bin\start-novel-reader.ps1 -Client none
```

不用 embedding，直接启动 Claude Code：

```powershell
.\bin\start-novel-reader.ps1 -NoEmbedding
```

指定模型路径：

```powershell
.\bin\start-novel-reader.ps1 -ModelPath "C:\Users\xsjhxs\.cache\modelscope\hub\models\Qwen\Qwen3-Embedding-0.6B"
```

首次成功后，模型路径和端口会保存到：

```text
.novel-reader-local\config.json
```

这个目录已加入 `.gitignore`，不会上传 GitHub。

下面是手动启动方式，通常不需要再手动操作。

先确认 `qwen_embed_server.py` 在插件目录里，并且端口是 `8081`。

启动服务：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:QWEN_EMBED_MODEL_PATH="C:\Users\xsjhxs\.cache\modelscope\hub\models\Qwen\Qwen3-Embedding-0.6B"
$env:QWEN_EMBED_BATCH="4"

python .\qwen_embed_server.py
```

看到下面这句后不要关闭窗口：

```text
Application startup complete.
```

测试服务：

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8081/v1/embeddings" `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"model":"qwen3-embedding-0.6b","input":["测试一段小说文本"]}'
```

## 4. 给小说建立语义索引

另开一个 PowerShell：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:NOVEL_READER_EMBED_BASE_URL="http://127.0.0.1:8081/v1"
$env:NOVEL_READER_EMBED_API_KEY="local"
$env:NOVEL_READER_EMBED_MODEL="qwen3-embedding-0.6b"

python .\bin\novel-reader embed 1-4-c52b6641a4 --provider openai-compatible --batch-size 4 --max-chars 1500
```

如果显存或 CPU 压力大，把 batch 降低：

```powershell
python .\bin\novel-reader embed 1-4-c52b6641a4 --provider openai-compatible --batch-size 2 --max-chars 1000
```

## 6. 续写功能

`continue` 不直接写正文，它生成“续写任务包”：前文上下文、剧情证据、风格证据、约束和自检清单。

从某章后续写：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --json
```

带续写大纲：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "路明非收到新的学院任务，线索指向一座废弃教堂" --scene 悬疑 --json
```

保存续写任务包：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "路明非收到新的学院任务，线索指向一座废弃教堂" --scene 悬疑 --write
```

输出位置：

```text
.novel-reader\1-4-c52b6641a4\continuations\continuation-pack-latest.md
.novel-reader\1-4-c52b6641a4\continuations\continuation-pack-latest.json
```

从具体 chunk 后续写：

```powershell
python .\bin\novel-reader read 1-4-c52b6641a4 --chapter 12
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chunk c0012-003 --outline "下一幕进入废弃教堂" --json
```

使用语义检索增强续写证据：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "废弃教堂里的龙类线索" --scene 悬疑 --semantic --json
```

## 7. 在 Claude Code 里使用

启动 Claude Code 前，先在同一个 PowerShell 设置 embedding 环境变量：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:NOVEL_READER_EMBED_BASE_URL="http://127.0.0.1:8081/v1"
$env:NOVEL_READER_EMBED_API_KEY="local"
$env:NOVEL_READER_EMBED_MODEL="qwen3-embedding-0.6b"

claude
```

Claude Code 里可以这样说：

```text
使用 novel-reader，查看 1-4-c52b6641a4 的阅读状态。
```

```text
使用 novel-reader，用语义检索回答：1-4-c52b6641a4 里某个伏笔在哪里回收？请引用章节和 chunk。
```

```text
使用 novel-reader，为 1-4-c52b6641a4 从第 12 章后生成续写任务包，大纲是：路明非收到新的学院任务，线索指向一座废弃教堂。场景类型用悬疑。
```

```text
根据刚才的续写任务包，写一段原创续写正文，正文后输出自检清单。
```

## 8. 注意事项

- `embed` 需要 Qwen 服务窗口一直开着。
- 如果 Claude Code 不能用语义检索，通常是启动 `claude` 前没有设置 `NOVEL_READER_EMBED_*`。
- `continue` 只生成任务包，不直接生成正文。
- 风格蒸馏和续写都默认是原创迁移，不复制原文，不生成直接仿冒作者的提示词。
- 如果 CPU/GPU 压力大，优先降低 `--batch-size` 和 `--max-chars`。
