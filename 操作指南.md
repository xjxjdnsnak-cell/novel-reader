# Novel Reader 操作指南

这份指南面向日常使用：导入小说、连接本地 Qwen embedding、在 Claude Code 里检索/问答/分析/续写。

## 1. 准备

插件目录：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader
```

查看已导入小说：

```powershell
python .\bin\novel-reader list
```

你已经导入的《龙族》：

```text
book_id: 1-4-c52b6641a4
title: 《龙族》（实体版1-4部全本）
```

## 2. 导入新小说

只支持 TXT / Markdown：

```powershell
python .\bin\novel-reader ingest "C:\Users\xsjhxs\Desktop\小说\新小说.txt"
```

导入成功后记下输出里的 `book_id`。

如果同一本书要重建索引：

```powershell
python .\bin\novel-reader ingest "C:\Users\xsjhxs\Desktop\小说\新小说.txt" --force
```

## 3. 启动本地 Qwen Embedding 服务

先确认 `qwen_embed_server.py` 在插件目录里，并且端口是 `8081`。

启动服务：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:QWEN_EMBED_MODEL_PATH="C:\Users\xsjhxs\.cache\modelscope\hub\models\Qwen\Qwen3-Embedding-0.6B"
$env:QWEN_EMBED_BATCH="4"

python .\qwen_embed_server.py
```

看到下面这句后不要关闭窗口：

```text
Application startup complete.
```

测试服务：

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8081/v1/embeddings" `
  -Method Post `
  -ContentType "application/json" `
  -Body '{"model":"qwen3-embedding-0.6b","input":["测试一段小说文本"]}'
```

## 4. 给小说建立语义索引

另开一个 PowerShell：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:NOVEL_READER_EMBED_BASE_URL="http://127.0.0.1:8081/v1"
$env:NOVEL_READER_EMBED_API_KEY="local"
$env:NOVEL_READER_EMBED_MODEL="qwen3-embedding-0.6b"

python .\bin\novel-reader embed 1-4-c52b6641a4 --provider openai-compatible --batch-size 4 --max-chars 1500
```

如果显存或 CPU 压力大，把 batch 降低：

```powershell
python .\bin\novel-reader embed 1-4-c52b6641a4 --provider openai-compatible --batch-size 2 --max-chars 1000
```

## 5. 常用命令

查看状态：

```powershell
python .\bin\novel-reader status 1-4-c52b6641a4
```

读取章节：

```powershell
python .\bin\novel-reader read 1-4-c52b6641a4 --chapter 12
```

关键词检索：

```powershell
python .\bin\novel-reader search 1-4-c52b6641a4 "路明非 第一次 发现 真相"
```

语义检索：

```powershell
python .\bin\novel-reader search 1-4-c52b6641a4 "路明非第一次意识到龙族世界的真相" --semantic
```

情节问答证据包：

```powershell
python .\bin\novel-reader ask 1-4-c52b6641a4 "某个伏笔在哪里回收？" --semantic
```

情节梳理：

```powershell
python .\bin\novel-reader outline 1-4-c52b6641a4 --write
```

全书地图：

```powershell
python .\bin\novel-reader map 1-4-c52b6641a4
```

写作分析：

```powershell
python .\bin\novel-reader analyze 1-4-c52b6641a4
```

语言风格蒸馏：

```powershell
python .\bin\novel-reader style 1-4-c52b6641a4 --write
```

按场景蒸馏风格：

```powershell
python .\bin\novel-reader style 1-4-c52b6641a4 --scene 悬疑 --json
```

## 6. 续写功能

`continue` 不直接写正文，它生成“续写任务包”：前文上下文、剧情证据、风格证据、约束和自检清单。

从某章后续写：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --json
```

带续写大纲：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "路明非收到新的学院任务，线索指向一座废弃教堂" --scene 悬疑 --json
```

保存续写任务包：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "路明非收到新的学院任务，线索指向一座废弃教堂" --scene 悬疑 --write
```

输出位置：

```text
.novel-reader\1-4-c52b6641a4\continuations\continuation-pack-latest.md
.novel-reader\1-4-c52b6641a4\continuations\continuation-pack-latest.json
```

从具体 chunk 后续写：

```powershell
python .\bin\novel-reader read 1-4-c52b6641a4 --chapter 12
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chunk c0012-003 --outline "下一幕进入废弃教堂" --json
```

使用语义检索增强续写证据：

```powershell
python .\bin\novel-reader continue 1-4-c52b6641a4 --after-chapter 12 --outline "废弃教堂里的龙类线索" --scene 悬疑 --semantic --json
```

## 7. 在 Claude Code 里使用

启动 Claude Code 前，先在同一个 PowerShell 设置 embedding 环境变量：

```powershell
cd C:\Users\xsjhxs\Desktop\Codex1\skills\novel-reader

$env:NOVEL_READER_EMBED_BASE_URL="http://127.0.0.1:8081/v1"
$env:NOVEL_READER_EMBED_API_KEY="local"
$env:NOVEL_READER_EMBED_MODEL="qwen3-embedding-0.6b"

claude
```

Claude Code 里可以这样说：

```text
使用 novel-reader，查看 1-4-c52b6641a4 的阅读状态。
```

```text
使用 novel-reader，用语义检索回答：1-4-c52b6641a4 里某个伏笔在哪里回收？请引用章节和 chunk。
```

```text
使用 novel-reader，为 1-4-c52b6641a4 从第 12 章后生成续写任务包，大纲是：路明非收到新的学院任务，线索指向一座废弃教堂。场景类型用悬疑。
```

```text
根据刚才的续写任务包，写一段原创续写正文，正文后输出自检清单。
```

## 8. 注意事项

- `embed` 需要 Qwen 服务窗口一直开着。
- 如果 Claude Code 不能用语义检索，通常是启动 `claude` 前没有设置 `NOVEL_READER_EMBED_*`。
- `continue` 只生成任务包，不直接生成正文。
- 风格蒸馏和续写都默认是原创迁移，不复制原文，不生成直接仿冒作者的提示词。
- 如果 CPU/GPU 压力大，优先降低 `--batch-size` 和 `--max-chars`。
